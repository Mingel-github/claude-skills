# 精读字段模板

## 核心字段（必提取）

### 1. 研究问题与背景
- 核心问题是什么
- 现有方法不足（具体指出Gap）
- 问题形式化定义（如有）

### 2. 核心方法
- 方法名称
- 全过程推演（像给非专业人士讲故事一样完整描述）
- 关键公式（LaTeX，见公式控制规则）
- 算法伪代码（如有）

### 3. 实验数据与性能指标
- 关键性能指标（精度/RMSE/成功率/计算时间等）
- 实验数据集/仿真场景设置
- 与其他方法的对比结果
- 消融实验（如有）

### 4. 创新点
- 3-5点核心贡献（每条一句话）

### 5. 局限与未来方向
- 方法局限
- 适用场景边界条件
- 未来工作

## 可选字段（根据参数启用）

### 6. 方法对比点评
- 相比同类方法的主要优势
- 主要局限或不足
- 适用场景的边界条件

### 7. 团队与机构信息
按详细程度分级：
- **brief**：第一作者+通讯作者+所在学校/学院
- **detailed**：+ 实验室名称、主攻方向、代表人物、标志性论文
- **genealogy**：+ 师承关系、合作与竞争关系

## 公式控制

| 级别 | 行为 |
|------|------|
| none | 全程禁止公式符号，用自然语言描述 |
| compact | 仅保留1-2个核心公式，每个附文字解释 |
| full | 保留完整数学推导，每个公式附文字解释 |

## 格式要求
- 输出用中文
- 公式必须LaTeX（`$...$`行内，`$$...$$`独立）
- 数值要精确，不四舍五入
- 不存在的部分写"未提及"
- 方法过程做"全过程推演"：按步骤完整描述，读者无需看原文也能理解
